Pipeline

From query to ranked answer in seconds

Six sequential stages, each powered by a specialized component. No hardcoded logic — the LLM drives routing, extraction, and narration.

01

Intent routing

Claude classifies your message into one of six intent types: shopping query, quick commerce, price check, comparison, recommendation, or general. Hindi and Hinglish work natively.

02

Structured extraction

Natural language is converted to a typed JSON — budget ceiling, brand preference, feature requirements, pincode — before any platform is touched.

03

Parallel search

All configured platforms are queried concurrently. Results are deduplicated by title fingerprint before scoring, eliminating near-duplicate listings.

04

6-factor ranking

Price fit, rating, review count, delivery speed, platform trust, and feature match are each weighted and combined into a 0–100 composite score.

05

Review summarisation

Claude generates concise pros, cons, and a verdict for each top-ranked product — concurrently, so latency stays flat regardless of how many products surface.

06

SSE streaming

Events flow to the frontend in real time: thinking badges, intent echo, product cards, then token-by-token AI narration — first token in under a second.

Architecture

Built to scale

🔀

LLM intent router

Claude classifies and routes every message. No brittle keyword lists — handles slang, Hinglish, and typos without special casing.

⚡

Async streaming

FastAPI + SSE: results stream the moment they're ready. No waiting for the full pipeline to complete before the UI updates.

🔒

Per-session isolation

UUID-keyed session objects replace the global shared memory bug. Every user gets an independent conversation context with TTL cache.

🎯

6-factor scorer

Price fit, rating, review volume, delivery speed, platform trust, feature match — weighted into a transparent composite score, not a black box.

♻️

Dedup engine

Title fingerprinting removes near-duplicate listings across platforms before they ever reach the ranker or the UI.

🔌

Plugin architecture

Add any new platform in four steps: extend BaseIntegration, implement async search(), register it, add env keys. Zero core changes.

Developer

Simple API

POST /api/chat SSE streaming

Send a natural-language shopping query and receive a stream of typed events.

{
  "message":    "best wireless headphones under ₹3000",
  "session_id": "abc123",   // optional
  "pincode":    "110001"    // optional, for delivery ETAs
}

// SSE events emitted:
event: thinking  → "Searching across platforms…"
event: intent    → { query_text, budget_max, … }
event: products  → { products: [...], platforms_searched: [...] }
event: text      → "Best pick is…"  // token-by-token
event: done      → { session_id }
GET /api/search Non-streaming

Synchronous search for server-to-server use cases.

/api/search?q=wireless+headphones&budget=3000&sort=best_value
GET /api/health Status check

Check which platforms are configured and whether the LLM is connected.

{
  "status":                "ok",
  "llm_connected":         true,
  "platforms_configured":  ["serp", "amazon", "flipkart"],
  "cache_connected":       false
}
